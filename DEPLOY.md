# 배포 가이드 — 스마트 통장

스마트 통장은 두 부분으로 나뉩니다.

- **앱 화면(index.html)** — 한 곳에 올려 두고 모든 선생님이 같은 주소로 씁니다. (GitHub Pages)
- **선생님별 구글 시트(Code.gs + Index)** — 선생님마다 자기 시트 사본을 하나씩 가집니다. 학생 데이터는 각자의 시트에 저장됩니다.

앱 화면은 선생님이 설정에 붙여넣은 **웹 앱 주소(.../exec)** 로만 그 선생님의 시트에 연결됩니다. 아무것도 연결하지 않으면 **체험 모드**(학생1~30, 이 기기에만 저장)로 바로 써볼 수 있습니다.

---

## A. 앱 화면 올리기 — 한 번만 (앱 관리자)

모든 선생님이 공유할 앱 주소를 만드는 단계입니다. 처음 한 번만 하면 됩니다.

1. GitHub에 새 저장소를 만들고 이 폴더의 **`index.html`** 을 올립니다. (이 파일 하나면 됩니다. 외부 CDN 없이 자체 완결됩니다 — 글꼴 링크만 예외)
2. 저장소 **Settings → Pages** 에서 브랜치를 지정해 게시합니다.
3. 잠시 뒤 `https://<계정>.github.io/<저장소>/` 같은 공용 주소가 생깁니다. 이 주소를 선생님들에게 안내합니다.

> 다른 정적 호스팅(Netlify, Cloudflare Pages 등)이나 학교 홈페이지에 올려도 됩니다. `index.html` 하나만 있으면 동작합니다.

---

## B. 선생님 시트 준비 — 선생님마다 (처음 한 번)

선생님이 자기 반 시트를 만들고 웹 앱으로 배포해 **웹 앱 주소(.../exec)** 를 받는 단계입니다. 자세한 안내는 시트를 처음 열면 나오는 **'사용 설명'** 탭에 그대로 들어 있습니다.

1. 준비된 **시트 사본**을 받습니다. (사본을 만들 때 Code.gs·Index 코드가 함께 따라옵니다)
   - 사본 배포용 링크는 시트 URL 끝의 `/edit` 를 `/copy` 로 바꾼 주소입니다.
   - 예) `https://docs.google.com/spreadsheets/d/<ID>/copy`
2. 시트에서 **확장 프로그램 → Apps Script** → 오른쪽 위 **배포 → 새 배포** → 톱니바퀴 **웹 앱**.
   - **다음 사용자 인증 정보로 실행**: 나(선생님 본인)
   - **액세스 권한이 있는 사용자**: 모든 사람 (또는 학교 Google 계정)
3. 처음이면 권한 승인 화면에서 본인 계정으로 허용합니다. (미확인 앱 경고가 뜨면 '고급 → 이동' 을 거칩니다)
4. 표시된 **웹 앱 URL** (`https://script.google.com/macros/s/.../exec`) 을 복사해 둡니다.

시트 탭(사용 설명·학생 명단·통계·용돈_기록·적립_기록·설정)은 처음 실행 때 자동으로 만들어집니다. 미리 만들려면 시트 상단 메뉴 **'🏦 스마트 통장' → 시트 초기화** 를 한 번 누르세요. `학생 명단` 탭은 A열이 **번호**, B열이 **이름**이며, 이름 열에 학생 이름을 한 줄씩 입력합니다. (적립통장의 활동을 저장하는 `적립_활동` 탭은 화면에서 숨겨져 있습니다.)

---

## C. 앱에 연결하고 학생용 주소 만들기 — 선생님

1. A에서 안내받은 **앱 공용 주소**를 브라우저에서 엽니다.
2. 통장 표지 아래 **⚙ 설정** 을 엽니다.
3. **웹 앱 주소** 칸에 B-4에서 복사한 `.../exec` 주소를 붙여넣고 **연결하기**.
   - 연결되면 이 반 명단을 바로 불러오고, 이 기기에는 다음부터 자동으로 이어집니다.
4. 그 아래 **학생용 입장 주소**가 만들어집니다. **복사** 버튼으로 복사해 학생들에게 공유합니다. (구글 클래스룸·문자·QR코드 등)
   - 학생용 주소 형식: `https://(앱 주소)/#c=<선생님 웹 앱 주소>`
   - 이 주소로 들어온 학생 화면에는 설정 버튼이 보이지 않습니다.

---

## 선생님이 할 일 — 대부분 시트에서

이 앱은 학생 화면이 중심입니다. 도장 찍기(확인)·설정 변경·통계 보기·공통 활동은 모두 시트에서 직접 합니다.

- **도장 찍기**: `적립_기록` / `용돈_기록` 시트의 **확인** 열 체크박스를 켜면 학생 화면에도 도장이 보입니다.
- **설정 변경**: `설정` 시트에서 은행이름·기능 사용 여부를 바꿉니다. 학생이 새로고침하면 반영됩니다.
- **학급 통계**: `통계` 시트에 학생별 합계가 자동 계산됩니다.
- 앱의 **⚙ 설정 → 선생님 화면 열기**(기본 비밀번호 `1234`)에서도 실적 확인·통계·시트 상태를 볼 수 있습니다.

---

## 체험 모드

앱 주소를 열고 아무것도 연결하지 않으면 체험 모드입니다.

- 명단은 **학생1~학생30**. 아무나 골라 바로 써볼 수 있습니다.
- 만든 내용은 **이 기기(브라우저)에만** 저장되어, 다음에 와도 이어서 볼 수 있습니다.
- 시트에 저장하는 기능은 동작하지 않고 "연결하면 저장돼요" 안내만 보여줍니다.
- ⚙ 설정에서 웹 앱 주소를 연결하면 그때부터 그 반 시트에 저장됩니다.

---

## 코드 수정 후 다시 배포

- **앱 화면(index.html)** 을 고쳤다면 GitHub Pages(또는 올린 곳)에 새 파일을 올립니다. 주소는 그대로, 모든 선생님이 자동으로 새 화면을 씁니다.
- **시트 코드(Code.gs)** 를 고쳤다면 각 선생님 시트에서 **배포 → 배포 관리 → 연필 → 새 버전 → 배포**. 웹 앱 주소는 그대로 유지됩니다.

## 자주 막히는 곳

- **연결에 실패해요**: 붙여넣은 주소가 `.../exec` 로 끝나는 웹 앱 주소가 맞는지, 배포의 액세스 권한이 '모든 사람'(또는 학교 계정)인지 확인하세요.
- **학생이 시트를 직접 못 보게 하고 싶어요**: 시트는 선생님만 접근 가능한 상태로 두고, 학생에게는 시트 URL이 아니라 **학생용 입장 주소**만 나눠주세요.
- **다른 선생님께도 나눠주고 싶어요**: 시트 URL 끝의 `/edit` 를 `/copy` 로 바꾼 링크를 보내면 됩니다. 받은 분은 자기 사본을 만들고 B·C 단계를 따라 하면 됩니다.
