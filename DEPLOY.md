# 배포 가이드 — Apps Script 웹앱

이 폴더의 파일을 Google Apps Script로 옮겨 웹앱으로 배포하면, 학생들은 URL 하나로 접속해서 사용할 수 있고 모든 데이터는 스프레드시트에 실시간으로 저장됩니다.

## 1) 스프레드시트 준비

1. 새 Google 스프레드시트를 만들어 적당한 이름(예: `2026 우리 반 통장`)으로 저장합니다.
2. 메뉴에서 **확장 프로그램 → Apps Script** 클릭. 새 Apps Script 프로젝트가 열립니다.

## 2) 코드 붙여넣기

Apps Script 편집기에 왼쪽 파일 목록이 있습니다. 두 개의 파일이 필요합니다.

### `Code.gs`
- 기본으로 만들어진 `Code.gs` 파일에, 이 폴더의 `Code.gs` 내용을 전체 복사해서 붙여넣고 저장합니다.

### `Index.html`
- 왼쪽 파일 목록 위의 **➕ → HTML** 을 눌러 새 HTML 파일을 만들고 이름을 정확히 **`Index`** 로 입력합니다. (확장자는 자동으로 `.html` 이 붙어요)
- 이 폴더의 `index.html` 내용을 전체 복사해서 새 `Index.html` 에 붙여넣고 저장합니다.

### (선택) `appsscript.json`
- Apps Script 편집기 → 톱니바퀴 ⚙ **프로젝트 설정** → "appsscript.json 매니페스트 파일을 편집기에 표시" 체크.
- 표시된 `appsscript.json` 내용을 이 폴더의 `appsscript.json` 으로 교체하면 시간대(서울)와 웹앱 기본 설정이 함께 들어갑니다.

## 3) 시트 탭 만들기 (선택, 자동으로도 됨)

처음 실행 시 자동으로 만들어지므로 건너뛰어도 됩니다. 미리 만들어두려면 Apps Script 편집기의 함수 드롭다운에서 `initSheets` 를 선택하고 **실행** 버튼을 누르세요. 권한 허용을 요청받으면 본인 계정으로 허용해 주세요.

성공하면 스프레드시트에 4개의 탭이 생깁니다:
- `명단` — 이름, 적립통장잠금, 용돈기입장잠금, 용돈기간시작, 용돈기간종료
- `적립_활동`
- `적립_기록`
- `용돈_기록`

`명단` 시트에 학생 이름을 한 줄씩 입력합니다. 잠금 칸은 이미 체크박스로 들어가 있으니 끄면 사용, 켜면 잠금.

## 4) 웹앱으로 배포

1. Apps Script 편집기 오른쪽 위 **배포 → 새 배포** 클릭
2. 톱니바퀴 ⚙ → **웹 앱** 선택
3. 설정:
   - **설명**: 아무거나 (예: "v1")
   - **다음 사용자 인증 정보로 실행**: **나(선생님 본인)**
   - **액세스 권한이 있는 사용자**: **모든 사람** (학생이 구글 로그인 없이 접속 가능) 또는 **Google 계정이 있는 모든 사용자** (학교 계정 강제)
4. **배포** 클릭 → 권한 허용
5. 표시된 **웹 앱 URL**(https://script.google.com/macros/s/.../exec) 을 복사 → 학생들에게 공유

> 액세스 권한이 "모든 사람"이면 URL을 아는 누구나 접속할 수 있으니, 학급 내에서만 공유하세요. 더 안전하게 가려면 "Google 계정이 있는 모든 사용자"로 두고 학교 계정을 사용하면 됩니다.

## 5) 처음 사용해보기

1. 학생들에게 공유한 URL로 접속.
2. **🧑‍🏫 선생님 모드** → 비밀번호 `1234` (선생님 모드에서 변경 가능)
3. **📊 시트 정보** 탭에서 **🛠️ 시트 초기화** 한 번 클릭 (필요한 탭/머리글 생성)
4. **📄 스프레드시트 열기** 버튼으로 시트를 열어 `명단` 탭에 학생 이름 입력
5. 앱으로 돌아와 **🔄 명단 다시 불러오기** 클릭 → 학생들이 들어올 준비 완료

## 6) 도장 찍기 — 두 가지 방법

- 앱 안에서: 선생님 모드 → **📋 실적 확인 & 도장** 탭 → 체크박스 켜기
- 시트에서: `적립_기록` 또는 `용돈_기록` 시트의 **확인** 열 체크박스를 직접 켜기

어느 쪽이든 학생이 자기 통장을 다시 열어보면 도장이 찍힌 모습이 보입니다.

## 코드 수정 후 다시 배포

코드를 수정했다면 **배포 → 배포 관리 → 연필 ✏ 아이콘 → 새 버전 → 배포**. URL은 그대로 유지됩니다.

## 자주 막히는 곳

- **로그인 화면에 "Apps Script 웹앱이 아닙니다" 가 뜸**: 웹앱 URL이 아니라 `file://` 로 열린 상태입니다. 위 4번에서 받은 `.../exec` URL로 들어가야 합니다.
- **권한 거부**: 첫 실행 시 권한 동의 화면이 막혀 보이면 "고급 → 안전하지 않음으로 이동" 을 거쳐야 할 수 있습니다 (Google에서 미확인 앱으로 표시되기 때문).
- **학생이 시트를 직접 보지 못하게 하고 싶음**: 시트 자체는 선생님만 접근 가능한 상태로 두고, Apps Script만 ALLOWED로 두세요. 학생들은 시트 URL을 받지 않고 웹앱 URL만 사용합니다.

## GitHub Pages 배포 (나중에)

지금 frontend는 Apps Script 안에 들어 있지만, 같은 `index.html` 을 GitHub Pages에 올려서 외부에서 Apps Script API를 호출하려면 두 가지를 추가 작업하면 됩니다:

1. `Code.gs` 에 `doPost(e)` 를 추가해서 JSON 요청을 라우팅
2. `index.html` 의 `callGS` 자리에서 `google.script.run` 대신 `fetch(APPS_SCRIPT_URL, {method:'POST', body:JSON.stringify({fn, args})})` 로 호출

필요한 시점에 알려주시면 추가 코드를 드릴게요.
